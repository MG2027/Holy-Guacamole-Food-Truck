LOTS OF FORMS!!!!!!

The form element contains a form on a web page. The <form> tag specifies the beginning of a form area. The closing </form> tag specifies the end of a form area. There can be multiple forms on a web page, but they CANNOT BE NESTED INSIDE ONE ANOTHER.

Input

The <input> configures a text box form control. The text box form control accepts text or numeric information such as names, e-mail addresses, phone numbers, and other text.

Email:

The type attribute declares what are acceptable inputs for this particular form. By giving type the value of text, you are instructing this form to only accept standard text. For example, someone could not upload a picture of their email address, they are forced to type it out.

The required attribute is new to HTML5 and makes it so that particular form is required to fill in before the user can progress forward. Think about any online form that you've filled out and if you missed a required piece of information it would inform you to go back and fill it in before you can progress.

Email:

The attribute type with value submit produces a button that when clicked will start the process of sending the inputted data to the desired location for the webmaster to make use of.

The attribute type with value reset produces a button that clears out the currently entered data.

From a style and visual perspective, if you want to put the label inside the box instead of outside set a placeholder attribute and then make it equal to what you want it to say.

Check Box

The check box form control allows the user to select one or more of a group of predetermined items. The input element with type="checkbox" configures a check box.

Radio Button

The radio button form control allows the user to select exactly one (and only one) choice from a group of predetermined items. Each radio button in a group is given the same name attribute and a unique value attribute. Because the name attribute is the same, the elements are identified as part of a group by the browsers and only one may be selected.

Scrolling Text Box

The scrolling text box form control accepts free-form comments, questions, or descriptions. The <textarea> tag denotes the beginning of the scrolling text box and <textarea> tag denotes the end of the scrolling text box.

The attributes cols and rows control height and width of your comment box. Cols is short for columns and this controls your width. A Cols="40" will allow that row to contain 40 characters (text and spaces) within that row. Rows represents the height of the comment box. The bigger the number the bigger the box.

Select Element

The select element contains and configures the select list form control. The <select> tag denotes the beginning of the select list. The closing </select> tag denotes the end of the select list.

the option element contains and configures the an option item displayed in the select list form control. The relationship between select and option for a list is similar to that of video and control. You start with a video tag and then encapsulate the source code within the video tag. The option tag also automatically starts the next tag on the next line, similar to a paragraph tag.

The size attribute for the select tag determines how many options are visible in the list without having to interact with it. If you want the entire list to show, increase the size value to the number of objects in your list.
